Preguntas

Internet y World Wide Web

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red física global de computadoras interconectadas, mientras que la World Wide Web es un servicio que funciona sobre Internet para compartir información mediante documentos HTML enlazados.

URLs y HTTP

¿Cuáles son las partes de un URL?

Las partes son: protocolo, dominio, puerto (opcional), ruta, parámetros (opcional), y fragmento (opcional).

¿Cuál es el propósito de los métodos HTTP?

  • GET: Obtener recursos
  • HEAD: Obtener metadatos del recurso
  • POST: Enviar datos para procesamiento
  • PUT: Actualizar un recurso completo
  • PATCH: Actualizar parcialmente un recurso
  • DELETE: Eliminar un recurso

Códigos HTTP y Responsabilidades

Código 200: Indica que la petición fue exitosa.

Error 404: Es responsabilidad del desarrollador corregirlo ya que indica que un recurso no existe en la ubicación solicitada.

Error 500: Es responsabilidad del desarrollador corregirlo ya que indica un error en el servidor.

CSS y Buenas Prácticas

¿Cuál es tu recomendación sobre el uso de !important en CSS?

Se debe evitar el uso de !important ya que rompe la especificidad natural de CSS y dificulta el mantenimiento del código. Solo debería usarse en casos muy específicos como sobrescribir estilos de librerías de terceros cuando no hay otra opción.

¿Por qué debe escogerse con cuidado una imagen de fondo en HTML?

Las imágenes de fondo deben escogerse cuidadosamente porque afectan la legibilidad del contenido, el tiempo de carga de la página, y el consumo de datos del usuario. Es importante considerar el contraste con el texto, el tamaño del archivo, y cómo se verá en diferentes tamaños de pantalla.

¿Cuál es tu recomendación al elegir unidades entre %, px y pt?

- Usar % para layouts responsivos y elementos que deben escalar con el viewport - Usar px para bordes, sombras y elementos que deben mantener tamaño consistente - Evitar pt ya que está diseñado para impresión y no pantallas

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

La versión minimizada del CSS mejora el rendimiento porque: - Reduce el tamaño del archivo eliminando espacios, comentarios y caracteres innecesarios - Disminuye el tiempo de descarga y el consumo de ancho de banda - Acelera el parsing del CSS por el navegador - Reduce la carga del servidor
