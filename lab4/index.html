Internet y World Wide Web

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red física global de computadoras interconectadas que permite la comunicación entre dispositivos, mientras que la World Wide Web es un servicio que funciona sobre Internet, siendo un sistema de documentos interconectados por hipervínculos que son accesibles a través de Internet usando un navegador web.

URLs y HTTP

¿Cuáles son las partes de un URL?

Las partes principales de un URL son: - Protocolo (http://, https://) - Dominio o hostname (www.ejemplo.com) - Puerto (opcional, ej: :80) - Ruta (/blog/articulo) - Parámetros de consulta (opcional, ej: ?id=123) - Fragmento (opcional, ej: #seccion)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicitar y recuperar datos del servidor
  • HEAD: Obtener los metadatos de un recurso sin obtener el contenido
  • POST: Enviar datos al servidor para crear nuevos recursos
  • PUT: Actualizar o reemplazar completamente un recurso existente
  • PATCH: Realizar modificaciones parciales a un recurso
  • DELETE: Eliminar un recurso específico

¿Qué método HTTP se debe utilizar al enviar un formulario HTML?

Se debe utilizar el método POST al enviar formularios HTML, especialmente para datos sensibles como usuarios y contraseñas. Esto es porque POST envía los datos en el cuerpo de la petición HTTP, no en la URL, lo que es más seguro y permite enviar mayor cantidad de datos.

¿Qué método HTTP se utiliza cuando se accede a una página a través de un URL?

Cuando se accede a una página web a través de un URL en el navegador, se utiliza el método GET. Este método es el predeterminado para solicitar recursos web y visualizar páginas.

Códigos HTTP y Responsabilidades

Código 200:

El código 200 significa "OK" e indica que la petición fue exitosa. No ocurrió ningún error y el servidor respondió correctamente con el recurso solicitado.

Error 404 - Responsabilidad del desarrollador:

Sí, es responsabilidad del desarrollador corregirlo ya que el error 404 (Not Found) indica que un recurso no existe en la ubicación solicitada. Esto puede ocurrir por enlaces rotos, recursos movidos o eliminados, y es importante mantener las rutas actualizadas y proporcionar redirecciones cuando sea necesario.

Error 500 - Responsabilidad del desarrollador:

Sí, es responsabilidad del desarrollador corregirlo ya que el error 500 (Internal Server Error) indica un problema en el servidor, generalmente causado por errores en el código del servidor, problemas de configuración o fallos en la base de datos que deben ser identificados y corregidos por el equipo de desarrollo.

HTML5

Elementos HTML desaprobados (deprecated):

Un atributo o elemento desaprobado significa que aunque aún puede funcionar, se considera obsoleto y podría dejar de ser soportado en futuras versiones. Algunos elementos desaprobados de HTML4 en HTML5 son: <center>, <font>, <strike>, <frame>, <frameset>, <noframes>, <acronym>, <applet>, <basefont>, <dir>, <isindex>

Diferencias principales entre HTML4 y HTML5:

  • Nuevos elementos semánticos: <header>, <nav>, <section>, <article>, <footer>
  • Mejor soporte para multimedia: <audio> y <video> nativos
  • Nuevos tipos de input y atributos para formularios
  • Canvas para gráficos
  • Almacenamiento local y APIs para aplicaciones web
  • Mejor manejo de errores de sintaxis

Componentes de estructura y estilo de una tabla:

  • Estructura: <table>, <tr>, <td>, <th>, <thead>, <tbody>, <tfoot>, <caption>
  • Estilo: border, border-collapse, padding, text-align, vertical-align, background-color, width, height

Principales controles de una forma HTML5:

  • Campos de texto: <input type="text">
  • Contraseñas: <input type="password">
  • Casillas de verificación: <input type="checkbox">
  • Botones de radio: <input type="radio">
  • Selectores: <select> y <option>
  • Áreas de texto: <textarea>
  • Botones: <button> o <input type="submit">
  • Nuevos en HTML5: email, number, date, color, range, search

CSS y Buenas Prácticas

¿Cuál es tu recomendación sobre el uso de !important en CSS?

Se debe evitar el uso de !important ya que rompe la especificidad natural de CSS y dificulta el mantenimiento del código. Solo debería usarse en casos muy específicos como sobrescribir estilos de librerías de terceros cuando no hay otra opción.

¿Por qué debe escogerse con cuidado una imagen de fondo en HTML?

Las imágenes de fondo deben escogerse cuidadosamente porque afectan la legibilidad del contenido, el tiempo de carga de la página, y el consumo de datos del usuario. Es importante considerar el contraste con el texto, el tamaño del archivo, y cómo se verá en diferentes tamaños de pantalla.

¿Cuál es tu recomendación al elegir unidades entre %, px y pt?

- Usar % para layouts responsivos y elementos que deben escalar con el viewport - Usar px para bordes, sombras y elementos que deben mantener tamaño consistente - Evitar pt ya que está diseñado para impresión y no pantallas

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

La versión minimizada del CSS mejora el rendimiento porque: - Reduce el tamaño del archivo eliminando espacios, comentarios y caracteres innecesarios - Disminuye el tiempo de descarga y el consumo de ancho de banda - Acelera el parsing del CSS por el navegador - Reduce la carga del servidor

JavaScript Fundamentals

¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Semejanzas:

  • Sintaxis básica similar (llaves, punto y coma)
  • Soportan programación orientada a objetos
  • Tienen bibliotecas estándar incorporadas

Diferencias:

  • JavaScript es un lenguaje interpretado, Java es compilado
  • JavaScript es de tipado dinámico, Java es de tipado estático
  • JavaScript se ejecuta principalmente en navegadores, Java es multiplataforma
  • JavaScript es prototípico, Java es basado en clases
  • JavaScript no necesita declaración de tipos, Java sí

¿Qué métodos tiene el objeto Date?

  • getDate(): Obtiene el día del mes (1-31)
  • getMonth(): Obtiene el mes (0-11)
  • getFullYear(): Obtiene el año completo
  • getHours(): Obtiene la hora (0-23)
  • getTime(): Obtiene milisegundos desde 1970
  • setDate(): Establece el día del mes
  • toLocaleString(): Convierte la fecha a string según la localización

¿Qué métodos tienen los arreglos?

  • push(): Añade elementos al final del arreglo
  • pop(): Elimina y devuelve el último elemento
  • map(): Crea un nuevo arreglo transformando cada elemento
  • filter(): Crea un nuevo arreglo con elementos que cumplen una condición
  • reduce(): Reduce el arreglo a un único valor
  • forEach(): Ejecuta una función para cada elemento
  • sort(): Ordena los elementos del arreglo

¿Cómo se declara una variable con alcance local dentro de una función?

Se utiliza la palabra clave 'let' o 'const' para declarar variables con alcance de bloque dentro de una función. Por ejemplo: function ejemplo() { let variableLocal = "solo visible aquí"; const constanteLocal = 42; }

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

  • Pueden ser modificadas por cualquier parte del código, causando efectos secundarios inesperados
  • Dificultan el testing y debugging del código
  • Pueden causar colisiones de nombres con otras variables
  • Aumentan el acoplamiento entre diferentes partes del código
  • Pueden causar problemas de seguridad al exponer datos sensibles
  • Dificultan el mantenimiento y la escalabilidad del código
